API Reference

ArcGIS Maps SDK for JavaScript v4.29 · Core classes used in these tutorials

This reference covers the six classes used across the tutorials. For the complete SDK API, see the official Esri API reference →

Map

The Map class holds the map's data model — its basemap and layers. It does not render anything itself; a MapView or SceneView does the rendering.

Module path: esri/Map

new Map(properties)
Constructor properties
PropertyTypeDescription
basemapString | BasemapThe basemap to display. Accepts a named string ("streets", "satellite", etc.) or a Basemap instance.
layers optionalCollection<Layer>Initial set of operational layers. Can also be added later via map.layers.add().
ground optionalString | GroundThe surface model. Used with SceneView. Pass "world-elevation" for a realistic terrain surface.
example
const map = new Map({
  basemap: "topo-vector",
  layers: [featureLayer]
});

MapView

Renders a Map in 2D. Controls the camera (zoom, centre, extent), handles user interaction, and provides methods for converting between screen and map coordinates.

Module path: esri/views/MapView

new MapView(properties)
Constructor properties
PropertyTypeDescription
containerString | HTMLElementThe ID of the DOM element (or the element itself) where the map renders.
mapMapThe Map instance to render.
zoom optionalNumberInitial zoom level. 0 = world, 23 = building level.
center optional[longitude, latitude]Initial map centre as [longitude, latitude].
extent optionalExtentInitial map extent. Overrides zoom and center if set.
ui optionalDefaultUIControls built-in UI components (compass, zoom buttons, attribution).
Key methods
MethodReturnsDescription
when(callback)PromiseFires when the view is ready to interact with. Use this before querying or manipulating layers.
goTo(target)PromiseAnimates the camera to a target (extent, graphic, geometry, or coordinate).
toMap(screenPoint)PointConverts a screen pixel coordinate to a map coordinate.
toScreen(mapPoint)ScreenPointConverts a map coordinate to a screen pixel coordinate.

FeatureLayer

Connects to a hosted ArcGIS feature service. Handles querying, rendering, popups, and editing. The most commonly used layer type.

Module path: esri/layers/FeatureLayer

new FeatureLayer(properties)
Constructor properties (most commonly used)
PropertyTypeDescription
urlStringURL to the feature service endpoint.
title optionalStringLayer title shown in the legend and LayerList.
visible optionalBooleanWhether the layer is visible on load. Default: true.
renderer optionalRendererControls how features are drawn. Accepts SimpleRenderer, UniqueValueRenderer, ClassBreaksRenderer.
popupTemplate optionalPopupTemplateDefines the popup that appears when a feature is clicked.
outFields optionalString[]Fields to load from the service. Use ["*"] for all. Restricting fields improves performance.
Key methods
MethodReturnsDescription
queryFeatures(query)Promise<FeatureSet>Executes a query and returns matching features.
queryFeatureCount(query)Promise<Number>Returns the count of features matching the query without fetching geometry or attributes.
createQuery()QueryCreates a Query object pre-populated with the layer's default query parameters.

GraphicsLayer

A client-side layer for displaying Graphic objects you create in code. Use it for user-drawn shapes, query result highlights, or any dynamic geometry.

Module path: esri/layers/GraphicsLayer

Key properties & methods
MemberTypeDescription
graphicsCollection<Graphic>The collection of all graphics in the layer. Use .add(), .remove(), .removeAll().
visibleBooleanToggle layer visibility.

Graphic

The basic visual unit. A Graphic has a geometry (where it is), a symbol (how it looks), attributes (data), and optionally a popup template.

Module path: esri/Graphic

new Graphic(properties)
PropertyTypeDescription
geometryGeometryThe shape: Point, Polyline, Polygon, or Extent.
symbolSymbolHow the geometry is drawn: SimpleMarkerSymbol, SimpleLineSymbol, SimpleFillSymbol, PictureMarkerSymbol.
attributes optionalObjectKey-value data attached to the graphic. Referenced in popup templates using {fieldName} syntax.
popupTemplate optionalPopupTemplatePopup shown when the graphic is clicked.

Query

Defines the parameters for a FeatureLayer.queryFeatures() call. Set attribute conditions, spatial filters, output fields, and pagination.

Module path: esri/rest/support/Query

new Query(properties)
PropertyTypeDescription
whereStringSQL WHERE clause. Example: "magnitude > 5 AND depth < 20". Use "1=1" to match all features.
geometry optionalGeometrySpatial filter geometry. Combined with spatialRelationship.
spatialRelationship optionalStringHow to apply the geometry filter. Options: "intersects" (default), "contains", "within", "touches", "overlaps".
outFieldsString[]Fields to return. ["*"] returns all. Limiting fields improves performance on large datasets.
returnGeometryBooleanWhether to return feature geometries. Set to true when you need to display results on the map.
num optionalNumberMax features to return per request. Default is the server's limit (often 1000 or 2000).
start optionalNumberZero-based offset for pagination. Use with num to page through results.
orderByFields optionalString[]Sort fields. Example: ["magnitude DESC"].
